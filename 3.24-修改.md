- 优化 `frontend/index.html` 与 `frontend/bank.html` 的页面头部、上传区、文案结构与信息标签。

- 缩小题库导入区面积，并将“解析文档”按钮改为更自然的长方形。
- 修复上传区超长文件名溢出，支持自动换行。

## 数学渲染与文本清洗

- 首页判题结果已要求后端尽量输出 KaTeX 可渲染的 LaTeX。
- 在 `frontend/static/js/app.js` 中增加结果文本清洗：
  - 去除 `学生作答`、`题目`、`综合点评` 前多余句号、圆点、项目符号。
  - 清理单独成行的无意义标点。
  - 将错误识别的空括号占位如 `\(\)`、`(\\)` 转为普通空括号。
- 在 `frontend/static/js/bank.js` 中增加对“裸 LaTeX”内容的自动包装，确保题库预览里的答案也能被 KaTeX 渲染。
- 题库导入预览中的题面与标准答案现在会在前端自动识别裸 LaTeX，并包装成可被 KaTeX 渲染的公式。

## 题库页表格与预览

- 为题库列表增加批量选择与批量删除按钮。
- 后端新增批量删除接口 `/api/questions/bulk-delete`。
- 题库列表与导入预览中的题面、答案改为折叠显示。
- 鼠标悬停或点击“展开”时，使用自定义数学预览浮层展示完整内容。
- 不再依赖浏览器原生 `title` 提示，因此预览内容支持数学渲染。

## PDF / 文档解析链路

- `backend/services/doc_parser.py` 现在主要负责抽取文档原始文本。
- 对 PDF 先尝试 `pdfplumber` 文本提取，若检测到乱码则自动回退为“页面转图片 + 多模态模型转写”。
- 早期为规则切题、页码清洗、答案串识别、题号匹配补过多轮修复；目前这些规则性逻辑已清理为冗余代码，不再作为主导导入方案。

## 模型主导的题库导入

- 新增 `backend/services/document_importer.py`，将题库导入改为模型主导流程：
  - 模型判断文件名属于试卷、答案或不明确归属。
  - 模型为同一套试卷/答案生成 `group_key`。
  - 模型直接从文档原始文本中抽取结构化题目列表。
  - 模型直接把试卷题目与答案条目合并为最终可入库题目。
- 题库导入现在是单上传框批量导入，不再区分“试卷按钮”和“答案按钮”。
- 只有答案文件、没有对应试卷时，不会写入题库预览。
- 只有试卷文件、没有答案时，会导入题目，但标准答案为空。
- 题目-答案匹配不再依赖旧的顺序配对或题号硬匹配，而是由模型直接合并为最终题库项。
- 文件归属判断也已交给模型处理，仅在模型返回异常时才回退到简单规则。

## 智能判题相关

- `backend/services/grading.py` 中增强了提示词：
  - 要求模型尽量输出规范 LaTeX。
  - 对空括号、填空横线等占位内容不要使用 LaTeX 定界符。
  - `recognized_content` 只能基于图片真实可见内容，不能把标准答案写进学生作答。
  - 若图片中学生未填写，必须输出“未作答”或“空白”。
  - 手动输入标准答案时，模型需要同时参考题目、学生作答和手动答案进行综合点评。
  - 如果模型自身判断与手动输入答案不一致，只能在点评中说明差异，但评分仍以手动输入答案为主。
- 增强了后端 JSON 解析容错：当模型输出中的 LaTeX 反斜杠破坏 JSON 时，先做修复再解析。
- `backend/services/question_matcher.py` 中的题库自动匹配已改为模型匹配，不再使用简单关键词重合逻辑。
- `backend/routers/judge.py` 中的手动判题与题库自动匹配判题都改为两段式：先纯看图片识别，再带标准答案评分，但保留第一次识别内容。
- 新增空白作答纠偏逻辑：如果第一次识别明确显示学生未作答，则第二次即使模型受标准答案干扰，也会强制判为 0 分并清空过程分。
- 优化空白纠偏逻辑：如果第二次识别中已经出现明确作答过程，则不再被第一次“空白”结论一刀切覆盖。
- 在综合点评前增加标准答案来源提示：
  - 手动输入模式会显示手动输入的标准答案，并说明“评分以手动输入为主”。
  - 题库自动匹配模式会显示题库标准答案。
  - 未匹配到题库标准答案时，会先提示“未在题库中搜索到可用标准答案”。

## 当前导入方案总结

- 当前题库导入采用“模型主导”的整条链路：
  1. 模型判断文件归属。
  2. 模型抽取试卷题目与答案条目。
  3. 模型完成题目与答案的最终合并。
  4. 结果进入预览并确认导入题库。

## 当前判题方案补充

- 手动输入标准答案与从题库自动匹配两条判题链路，都会先执行一次“只看图片”的识别。
- 若随后需要带标准答案重新评分，只复用评分结果，不复用第二次模型生成的识别文本。
- 对“学生作答为空白”的场景，后端会优先相信第一次图片识别结果，避免出现“图片空白但满分”的错误判断。
- 若第二次评分结果中已经出现明确学生作答内容，则会保留该作答识别，避免把真实作答误判成空白。
- 手动输入标准答案时，综合点评会把手动答案一并传给模型，并要求模型指出“模型判断”与“手动输入答案”是否一致，但最终评分仍以手动输入为主。
