# 要求日志

## 2026-03-18

（1）填空题，能够精准识别，并且判断正误。（不给答案自主判断）
（2）如果给定了答案，能够根据答案，进一步调整判断正误，标准答案可能有多种不同的写法，
（3）简答题，能够精准识别，并且判断正误。（不给答案自主判断）
（4）如果给定了答案，能够根据答案，进一步调整判断正误，并且能够嵌入。
（5）只能拍照，然后通过拍照上传后，进行识别与判断正误，还有打分，计算题要判断出过程分。
（6）尽量免费，通过调用国产大模型去做。



尽量确保项目在3月31日前交付。过程中的所有方案迭代与关键决策，请同步做好记录归档。每天把做完的内容和记录的文档发我一下。先做的是：高等数学（下）的填空题+计算题的拍照后的精准识别与评分（计算题包括步骤分），尽量低成本或免费。

## 2026-03-20

不要以1000份试卷来计算具体花费。

以：1门课程，1个人，1周30道数学填空题，10道数学计算题，无选择题目，一年共36周，全校1万人计算。

以：尽可能免费/便宜，而且方案执行到后期是可以正式上线使用为目标。

可以考虑以下2个方案：
充一个账号全部人使用，或者每个人用自己的号，用免费额度。
目的是尽可能价格低廉或免费，不使用“梯子”，全套方案执行后的产品可正式上线，在全校内部使用。



对于计算题来讲，如果提供了答案。最好能够让大模型去核对哪些步骤对了，哪些步骤错了，既要给出每个步骤点的评分具体依据（可以让老师们校验，具有可解释性和可交互性），也要给出最终评分（累加就行）。



你们还可以思考一下是否需要设计比较好的提示词，来约束大模型正确率更高，解释更加详细，另一方面站在使用者角度思考一下，怎么样提高他们使用的依赖性和体验感。

## 2026-03-21

填空题和计算题均需提供两种版本：
（1）若老师提供答案，由大模型比对所给答案并自行判断后打分；
（2）若老师不提供答案，直接由大模型进行判断。

此外，还需考虑：对于已给出答案的文档（如 PDF 或 Word），如何将其封装，以便大模型调用并对照答案进行判断，同时验证计算过程是否正确。

要实现的效果：数学老师上传一个包含题目与答案的1个或多个Word文档，程序能实现把题目和答案对应起来，调用大模型时就可以找到对应题目的答案去比对。让设定的题库具有可增加性，老师可以通过简单的手段去把题目和答案输入进去，然后就可以使用了，即大模型能找到对应题目的答案进行比对。

整体操作应力求简洁，避免冗余步骤。
